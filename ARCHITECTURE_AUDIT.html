Executive Summary

This platform is a purpose-built Kenya-market HRMS — not a general ERP. It handles the full employee lifecycle from recruitment to exit, with payroll disbursement via three live payment gateways, biometric attendance, geofencing, multi-approver workflows, and e-signatures via DocuSeal. The architecture is sound: multi-tenant, RBAC-governed, audit-logged, and event-driven via Celery.

The system is approximately 70% of the way to a commercially shippable SME HR platform for East Africa. The two most significant gaps are: (1) the Recruitment module is structurally weak relative to everything else — no dashboard UI, no interview scheduling, no ATS workflow beyond stage tagging; and (2) there is no analytics layer — the data is rich but nothing surfaces it to decision-makers.

Closest commercial competitor: Workpay.africa. Differentiation is the depth of compliance (statutory rates versioning, exit clearance, minimum wage alerts), the M-of-N payroll approval workflow, and the biometric attendance stack.

Role Hierarchy

Roles are ranked numerically — lower rank = more authority. Roles are system-defined or company-scoped. Deployed roles (25/35) are additionally scoped by StaffAssignment to restrict visible employees.

white_collar_employee40 Own Payslip Own Requests Not Required No Own Only No No
blue_collar_employee45 Own Payslip Own Requests Check In/Out No Own Only No No
Architecture Notes
  • PayrollHROnly is a hardcoded rule — no RBAC configuration can open payroll to managers or employees. Correct by design.
  • Deployed roles (25, 35) are scoped via the StaffAssignment table — they only see their assigned employees.
  • Legacy slugs hr / manager / employee map to internal_ variants for backward compatibility.
  • RBAC_STRICT=False by default — service-key calls bypass header checks. Should be True in production.

Key Business Rules

  • BackgroundCheck has a check constraint: either employee_id OR candidate_id — never both, never neither.
  • PayrollRun.status = paid is terminal — triggers document locking, no reversal path exists in code.
  • StatutoryRate.effective(kind, on_date) respects company overrides before global rates.
  • LeaveRequest and EmployeeProfile use db_table overrides to mirror Supabase — Django is secondary.
  • AttendanceEvent is a TimescaleDB hypertable partitioned on time when TIMESCALE_ENABLED=True.

Risk Assessment

🔴 Critical

Critical
SmileID Demo Mode ON by Default
SMILEID_DEMO_MODE=True — biometric verification always returns {verified: True, confidence: 0.99}. Anyone can clock in for anyone in production unless explicitly overridden.
Critical
Celery Always-Eager in Production
CELERY_TASK_ALWAYS_EAGER=True default — notifications, document generation, and compliance checks block the HTTP request thread. Will cause timeouts under load.
Critical
RBAC_STRICT=False by Default
Any caller with the service key can impersonate any user without providing role headers. All audited as rbac.legacy_access — masking real access patterns in the audit log.
Critical
face-api.js Score Threshold Bias
Default scoreThreshold: 0.5 caused systematic false rejections for darker skin tones. Patched to 0.3 / inputSize 608 on 24 June 2026.

🟠 High

High
No Payroll Reversal Path
Once status=paid, documents are locked. A wrong disbursement requires direct database intervention. No correction workflow exists.
High
Recruitment → Employee Handoff is Manual
A hired candidate never automatically becomes an EmployeeProfile. Data must be re-entered, introducing errors and delays in every hire.
High
DocuSeal Demo Mode ON by Default
DOCUSEAL_DEMO_MODE=True default — payroll approval e-signatures return stub submissions. No real signature is captured unless explicitly enabled in production config.
High
Supabase Mirror Fragility
LeaveRequest and EmployeeProfile are Supabase mirrors. Any Supabase schema migration silently breaks Django. No sync validation or migration guard exists.

🟡 Medium

Medium
Swahili Feature Advertised but Broken
Language toggle and t() hook exist; all strings remain English. The feature is visible to users but non-functional.
Medium
Client-Side jsPDF Payslip
Payslip PDF generated in browser. Fails silently on weak devices. Server-side ReportLab generation already exists — this is redundant and unreliable.
Medium
S3 Credential Exposure Risk
Supabase S3 keys in environment variables. Download endpoint lacks presigned URL pattern — if keys leak, entire hr-media bucket is accessible.
Medium
PesaPal Sandbox Default
PESAPAL_SANDBOX=True by default. A production deployment with this unset sends money to a test environment silently.

🟢 Low

Low
GROQ Model Hardcoded
llama3-70b-8192 — if deprecated by Groq, CV scoring silently fails with no fallback.
Low
Django Admin Without MFA
Admin URL obfuscated via env var (good). Still uses session auth without MFA — one credential leak = full admin access.

Competitive Moats

Three things that are genuinely impressive and must be protected
01
M-of-N Payroll Approval Workflow
Multi-approver quorum with DocuSeal + one-tap + dashboard converging to the same approval records. More sophisticated than anything in the Kenya SME space. Expensive to replicate.
02
Statutory Compliance Depth
Versioned statutory rates, minimum wage monitoring, exit clearance with 5 department sign-offs, disciplinary escalation chains. This is the hardest part of Kenya HR to get right — and it's done.
03
Payment Gateway Redundancy
Three independent disbursement paths (Daraja, IntaSend, PesaPal) covering M-Pesa, Airtel Money, and Bank EFT. Payroll disbursement never fails due to a single provider outage.
The Single Biggest Gap

The absence of an HR-admin-facing interface is not a feature gap — it is a distribution gap. Every module's backend investment is locked behind a raw API. Fix that first. Everything else follows.
